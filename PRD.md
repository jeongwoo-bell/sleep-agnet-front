# Sleep Agent — 웹 프론트엔드 PRD

## 1. 제품 개요

### 1.1 한 줄 요약

비개발자가 자연어로 랜딩페이지를 수정하고, 실시간으로 진행상황을 확인하며, 배포까지 완료하는 웹 채팅 인터페이스.

### 1.2 비전

```
Phase 1 (MVP)  : SleepThera 랜딩페이지 한정, 채팅 기반 코드 수정 + 배포
Phase 2        : 채팅 기록 저장, 브랜치 현황, PR 관리, 피그마 딥 연동
Phase 3        : 사내 전체 프로젝트 확장, RAG 기반 사내 지식 봇으로 진화
```

### 1.3 핵심 가치

| 기존 (슬랙 봇)             | 개선 (웹 UI)                      |
| -------------------------- | --------------------------------- |
| 이모지로 선택 (최대 9개)   | 버튼, 체크박스, 드래그앤드롭      |
| 텍스트 기반 진행상황       | 실시간 프로그레스 바 + 애니메이션 |
| 스레드에서만 대화 히스토리 | 채팅 기록 영구 저장 + 검색        |
| mrkdwn 포맷 제한           | 코드 diff 뷰어, 이미지 프리뷰     |
| 슬랙 계정 필요             | 사내 누구나 접근 가능             |

## 2. 타겟 사용자

### 2.1 Primary — 디자이너

- 피그마에서 디자인 완료 후, 링크만 붙여넣으면 구현 + 배포
- "Hero 섹션 타이틀 크기 키워줘" 같은 자연어로 직접 수정
- 코드/터미널을 몰라도 랜딩페이지에 기여 가능

### 2.2 Secondary — 기획자/마케터

- 카피 변경, FAQ 수정 등 텍스트 기반 수정
- "CTA 버튼 문구를 '지금 시작하기'로 바꿔줘"

### 2.3 Tertiary — 개발자

- 빠른 UI 수정이 필요할 때 채팅으로 요청
- PR 생성, 브랜치 관리 등 개발 워크플로우 연결

## 3. MVP 기능 명세 (Phase 1)

### 3.1 채팅 인터페이스

#### 3.1.1 메시지 입력

- 텍스트 입력 필드 (여러 줄 지원)
- 피그마 링크 붙여넣기 시 자동 감지 + 프리뷰 카드 표시
- 이미지/스크린샷 드래그앤드롭 첨부 (Phase 2)
- Enter 전송, Shift+Enter 줄바꿈

#### 3.1.2 메시지 표시

- 사용자 메시지: 우측 정렬
- 봇 응답: 좌측 정렬
  - 일반 텍스트 (talk, ask 응답)
  - 코드 변경 요약 (마크다운 렌더링)
  - 프리뷰 URL 카드 (클릭 시 새 탭)
  - 에러 메시지 (빨간 배경)

#### 3.1.3 대화 관리

- 새 대화 시작 버튼
- 같은 대화 내 후속 메시지 → 같은 브랜치에서 이어서 작업 (threadId 유지)

### 3.2 실시간 진행상황

요청 처리 중 WebSocket으로 받은 이벤트를 시각적으로 표시.

#### 3.2.1 프로그레스 스텝

```
① 요청 분석      → 메시지 분류 중
② 스펙 문서 확인  → 관련 스펙 검색 + 구현 여부 확인
③ 파일 분석      → 관련 파일 특정
④ 코드 수정      → AI가 코드 수정 중
⑤ 빌드 검증      → pnpm build 실행 중
⑥ 배포           → Vercel 배포 중
```

#### 3.2.2 UI 표현

- 각 스텝: 원형 아이콘 + 레이블 + 상태 (대기/진행/완료/실패)
- 현재 진행 중인 스텝: 펄스 애니메이션 + 스피너
- 완료된 스텝: 체크마크 + 초록색
- 실패한 스텝: X마크 + 빨간색 + 에러 메시지 툴팁
- 전체 진행률 바 (상단)
- 스텝 간 전환 시 부드러운 슬라이드/페이드 애니메이션

#### 3.2.3 완료 시

- 변경 요약 (마크다운)
- 프리뷰 URL 버튼 (클릭 → 새 탭에서 Vercel 프리뷰)
- "추가 수정하기" → 같은 대화에서 후속 요청
- "PR 만들기" 버튼 (Phase 2)

### 3.3 API 연동

백엔드: `st-design-agent` 레포의 `server.js` (Express + WebSocket)

#### 3.3.1 REST API

```
POST /api/request
  Body: { message, threadId?, figmaUrl?, userName?, chatHistory? }
  Response: { requestId, type, summary?, previewUrl?, branchName?, ... }

GET /api/health
  Response: { status: "ok" }
```

#### 3.3.2 WebSocket

```
ws://host:3001

Client → Server: { type: "subscribe", requestId: "uuid" }
Server → Client: { type: "progress", data: { step, state, ... } }
Server → Client: { type: "complete", data: { ... } }
Server → Client: { type: "error", data: { error: "..." } }
```

## 5. 기술 스택 (프론트엔드)

| 항목       | 선택                       | 이유                              |
| ---------- | -------------------------- | --------------------------------- |
| 프레임워크 | React (Vite)               | SPA 채팅앱, SSR 불필요, 빠른 빌드 |
| 스타일     | Tailwind CSS               | 사내 표준, 빠른 UI 구현           |
| 상태 관리  | zustand 또는 React Context | 채팅 상태 관리 단순               |
| WebSocket  | 네이티브 WebSocket API     | 별도 라이브러리 불필요            |
| 마크다운   | react-markdown             | 봇 응답 렌더링                    |
| 애니메이션 | framer-motion              | 프로그레스 애니메이션             |
| 배포       | Vercel                     | 사내 표준                         |

## 6. Phase 2 기능 (MVP 이후)

### 6.1 채팅 기록

- 대화 목록 사이드바 (최근순)
- 대화별 제목 자동 생성 (첫 메시지 기반)
- 대화 검색
- DB 저장 (Supabase 또는 SQLite)

### 6.2 브랜치 현황

- 현재 활성 브랜치 표시
- 브랜치별 변경 이력
- PR 상태 표시 (open/merged/closed)

### 6.3 PR 관리

- "PR 만들기" 버튼 → Draft PR 자동 생성
- PR 링크 표시

### 6.4 피그마 딥 연동

- 피그마 링크 붙이면 디자인 임베드 프리뷰
- 디자인 ↔ 코드 변경 비교 뷰

### 6.5 인증

- 사내 SSO 또는 간단한 비밀번호 게이트
- 사용자별 대화 기록 분리

## 7. Phase 3 기능 (장기 목표)

### 7.1 사내 RAG 봇

- 채팅 기록 + docs 레포 + 코드베이스를 벡터 DB에 인덱싱
- Haiku 대신 사내 RAG 봇으로 자연어 분류/분석
- 사내 컨텍스트 기반 더 정확한 판단

### 7.2 멀티 프로젝트

- `REPO_URL`을 프로젝트별로 전환
- 프로젝트 선택 UI
- 프로젝트별 docs/스펙 자동 연결

### 7.3 실시간 코드 diff

- 변경 전/후 코드 diff 뷰어
- 변경 사항 승인/거부 기능

## 8. MVP 개발 순서

```
1. 프로젝트 세팅 (Next.js + Tailwind + framer-motion)
2. 채팅 UI 컴포넌트 (입력, 메시지 버블, 스크롤)
3. API 연동 (POST /api/request)
4. WebSocket 연결 + 실시간 진행상황 표시
5. 프로그레스 컴포넌트 (스텝별 상태 + 애니메이션)
6. 완료 화면 (변경 요약 + 프리뷰 링크)
7. 에러 처리 + 로딩 상태
8. 배포 (Vercel)
```
